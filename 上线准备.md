## 下一步推荐顺序

### 第一步：继续补作品，但控制资源习惯

- 后续新加视频时，尽量准备一份网页展示版
- 后续新加详情图时，优先放展示尺寸图，不直接塞原始导出大图

### 第二步：上线前做补充减重

这一轮已经把主要大图压过一遍了。后面如果继续补充新项目，优先检查这些目录：

- `assets/projects/motion`
- `assets/projects/ai`
- `assets/projects/graphic/h5`
- 新增的品牌详情图目录

建议目标：

- 详情长图单张尽量压到 `1MB - 4MB`
- 缩略图尽量压到 `200KB - 600KB`
- 网页展示优先 `webp` 或优化后的 `jpg/png`

### 第三步：上线前确认展示范围

- 还没准备好的项目继续隐藏
- 检查每个项目是否都有：
  - 封面图
  - 详情图目录
  - 正确标题
  - 正确外链（如果需要）

### 第四步：正式上线

建议路线：

1. 上传到 GitHub
2. 接入 Vercel 免费方案
3. 先发一个预览链接自查
4. 没问题再作为面试展示链接使用

## 当前适合的策略

你现在最合适的方式不是立刻做“最终上线包”，而是：

- 继续补内容
- 我们保持页面结构稳定
- 真正上线前再做一次资源瘦身
- 最后再接 Vercel

这样效率最高，也不会因为现在反复压资源影响你继续整理作品。
